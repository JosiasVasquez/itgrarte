Cuarta sesión curso de control de versiones con git y github:

Temas vistos en la sesión:

  • Repaso
  • Qué es GitHub

    GitHub es una plataforma de desarrollo colaborativo de software, donde se pueden alojar proyectos utilizando el sistema de control de versiones Git. Permite la colaboración entre múltiples desarrolladores y la gestión de proyectos de software.

  • Exploración por la interfaz de GitHub

    En esta sección se explora la interfaz de GitHub, entendiendo las distintas secciones y funcionalidades que ofrece la plataforma para la gestión y colaboración en proyectos.

  • Exploración y edición de perfil en GitHub

    Aquí se enseñan las maneras de explorar y editar el perfil del usuario en GitHub, incluyendo cómo agregar información personal, cambiar avatar, y configurar preferencias de cuenta.

  • Páginas y acciones de los repositorios en GitHub
    • Fork - Hacer una copia de un repositorio en la propia cuenta de GitHub.
    • Star - Marcar un repositorio como favorito.
    • Add this repository to a list - Añadir el repositorio a una lista personalizada.
    • Code - Exploración del código fuente del repositorio.
    • Issues - Gestión de incidencias y tareas pendientes.
    • Pull requests - Propuestas de cambios al código.
    • Discussions - Espacio para discusiones y comentarios del proyecto.
    • Actions - Automatización de flujos de trabajo.
    • Wiki - Documentación del proyecto organizada en páginas.
    • Security - Configuración y monitorización de la seguridad del repositorio.
    • Insights - Estadísticas y análisis del repositorio.
  • Contenido de la página Code de un repositorio

    La pestaña "Code" contiene todo el árbol de directorios del proyecto, archivos, commits recientes, y botones para clonar o descargar el repositorio.

  • El archivo readme.md

    Un archivo `README.md` proporciona una descripción del proyecto. Es comúnmente utilizado para explicar qué hace el proyecto, cómo configurarlo y ejecutarlo, entre otras cosas.

  • Como clonar un repositorio por HTTPS y SSH

    Clonar un repositorio significa descargar una copia del repositorio remoto a tu máquina local. Se puede hacer usando URLs HTTPS o claves SSH según la configuración de seguridad y preferencias.

  • Cómo crear una clave SSH

    Las claves SSH se crean para habilitar una conexión segura entre las máquinas locales y los servidores. Es una medida de seguridad preferida sobre HTTPS para operaciones de escritura.

  • Cómo agregar la clave SSH a nuestra cuenta en GitHub

    Después de crear una clave SSH, es necesario añadirla a tu cuenta de GitHub para que pueda reconocerla y permitir operaciones de escritura desde tu máquina local.

  • Archivo config en la carpeta .ssh

    El archivo config en la carpeta .ssh es un archivo de configuración que permite personalizar la conexión SSH para diferentes hosts o servidores. En este archivo, se pueden especificar parámetros como el nombre del host, el puerto, la clave de autenticación y el usuario.

    Sintaxis para GitHub por el puerto 443

    A continuación, se muestra la sintaxis para configurar la conexión SSH para GitHub utilizando el puerto 443:

    			Host github
			HostName ssh.github.com
			Port 443
			IdentityFile ~/.ssh/id_ed25519
			User git

    En esta configuración:

    • Host: Especifica el nombre del host que se utilizará para la conexión SSH. En este caso, se utiliza "github", Al clonar, o buscar la conexión se utiliza para hacer referencia al hostname, por ejemplo en lugar de github.com o ssh.github.com se usa solo github porque así se estableció el host, esto se puede cambiar como se desee. Por ejemplo para comprobar la conexión se escribiría solo ssh github.
    • HostName: Especifica el nombre real del host al que se conectará. En este caso, se utiliza "ssh.github.com".
    • Port: Especifica el puerto que se utilizará para la conexión SSH. En este caso, se utiliza el puerto 443.
    • IdentityFile: Especifica la ruta a la clave de autenticación que se utilizará para la conexión SSH. En este caso, se utiliza la clave "~/.ssh/id_ed25519".
    • User: Especifica el usuario que se utilizará para la conexión SSH. En este caso, se utiliza el usuario "git".

    Al utilizar esta configuración, se puede establecer una conexión SSH con GitHub utilizando el comando "ssh github" en lugar de "ssh git@github.com". esto porque en la configuración ya se indicó el usuario, el puerto y el hostname y solo con el host se hacer referencia a él.

  • Cómo agregar la clave GPG a nuestra cuenta en GitHub

    Las claves GPG se utilizan para firmar commits, lo que añade una capa de verificación de identidad. Se debe añadir la clave pública GPG a tu cuenta de GitHub.

  • Subir cambios del repositorio local al repositorio remoto

    Esta acción consiste en enviar los commits realizados en el repositorio local al repositorio remoto usando el comando `git push`.

Comandos vistos en la sesión:

gpg --export -a "Nombre de Usuario" > public.key
Este comando exporta la clave pública GPG del usuario especificado a un archivo llamado `public.key`.
ssh-keygen -t ed25519 -C "correo@ejemplo.com"
Genera una nueva clave SSH utilizando el algoritmo Ed25519. El correo electrónico suele ser usado como comentario para identificar la clave.
ssh git@github.com
Este comando comprueba la conexión SSH a GitHub. Si la configuración es correcta, debe devolver un mensaje de bienvenida de GitHub.
ssh -p 443 git@ssh.github.com
Este comando comprueba la conexión SSH a GitHub en el puerto 443, alternativa cuando el comando anterior no funciona. Si la configuración es correcta, debe devolver un mensaje de bienvenida de GitHub.
git clone https://dominio.com/usuario/repositorio.git
Clona un repositorio de GitHub utilizando la URL HTTPS.
git clone git@github.com:usuario/repositorio
Clona un repositorio de GitHub utilizando la URL SSH.
git push
Envía los cambios realizados en el repositorio local al repositorio remoto.
